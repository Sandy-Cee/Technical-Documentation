What is CSS?

Like HTML, CSS (Cascading Style Sheets) is not a programming language. It’s not a markup language either. CSS is a style sheet language. CSS is used to selectively style HTML elements.


CSS can be used for very basic document text styling — for example, for changing the color and size of headings and links. It can be used to create a layout — for example, turning a single column of text into a layout with a main content area and a sidebar for related information. It can even be used for effects such as animation.

How to add CSS

When a browser reads a style sheet, it will format the HTML document according to the information in the style sheet.

Three Ways to Insert CSS

  • External CSS
  • Internal CSS
  • Inline CSS

Priority order -- Inline > Internal > External


External CSS

With an external style sheet, the look of an entire website can be altered by changing just one file. Each HTML page must include a reference to the external style sheet file inside the <link> element, inside the <head> section. An external style sheet can be written in any text editor and must be saved with a .css extension. The external .css file should not contain any HTML tags.

Internal CSS

An internal style sheet may be used if one single HTML page has a unique style. The internal style is defined within the <style> element, inside the <head> section of an HTML page.

Inline CSS

An inline style may be used to apply a unique style for a single element. To use inline styles, the <style> attribute is added to the relevant element. The style attribute can contain any CSS property.


Tip: An inline style loses many of the advantages of a style sheet (by mixing content with presentation). This method should be used sparingly.

CSS syntax

CSS is a rule-based language — rules are defined by specifying groups of styles that should be applied to individual elements or groups of elements on a web page.


For example, the main heading on the web page can be shown as large red text. The following code shows a very simple CSS rule that would achieve the styling described above:

      	h1       { color: red; font-size: 5em; }
selector {property:value;property:value}
  • In the above example, the CSS ruleset opens with a selector. This selects the HTML element to be styled. In this case, styling level one headings (<h1>).
  • Then a set of curly braces are added { }.
  • Inside the braces will be one or more declarations, which take the form of property and property value pairs. The property is specified (color in the above example) before the colon, and then the value of the property after the colon (red in this example).
  • This example contains two declarations, one for color and the other for font-size. Each pair specifies a property of the element(s) selected (<h1>) in this case), then a value desired for the property.

CSS properties have different allowable values, depending on which property is being specified. In the example, there is the color property, which can take various color values. There is also the font-size property. This property can take various size units as a value (use "em" and "rem" units as they are both scalable; "px" is not).


A CSS stylesheet will contain many such rules, written one after the other:


h1 { color: red; font-size: 5em; } 
p { color: black; }

Multiple elements can also be selected and then a single ruleset applied to all of them at once. Multiple selectors are separated by commas. For example:

  
p, li, h1 { color: red }
CSS selectors

CSS selectors are used to "find" (or select) the HTML elements to be styled.

  • Three simple selectors:
    • Element Selector
    • Id Selector
    • Class Selector

Priority of Selectors - Id > Class > Element


Element Selector

This selector is used to select HTML elements by its name. For example:

	
h1 { color: red; }

The heading tag was selected to change the color property i.e., text color, to red. Now whatever is written in this tag (content) will have the text color as red.

ID Selector

This selector is used to target specific or unique elements that have been given an “id” in html. For example:

	
#unique { color:red; }

The id tag was selected to change the color property i.e., text color, to red. Now whatever is written in this tag (content) will have the text color as red.

Class Selector

This selector is used to target a specific class of the html element. For example:

	
.group { color: red; }

The class tag was selected to change the color property i.e., text color, to red. Now whatever is written in this tag (content) will have the text color as red.

CSS colors

There are different colouring schemes in CSS, with the main three as follows:

  • RGB
    • This starts with rgb and takes 3 parameters, corresponding to red, green and blue, with each parameter value varying from 0 to 255. For example, rgb(255, 0, 0); means the color red.
  • HEX
    • Hex code starts with # and comprises of 6 numbers which are further divided into 3 sets of 2 numbers, corresponding to red, green, and blue, with the single set values varying from 00 to 99 and AA to FF. For example, #ff0000; means the color red.
  • RGBA
    • This starts with rgb and takes 4 parameters, corresponding to red, green, blue, and alpha (this a value represents the level of transparency/opacity of the color), with the value of the first three parameters varying from 0 to 255, and the last parameter ranging from 0.0 to 1.0; 0.0 being fully transparent, 1.0 being fully opaque. For example, rgba(255, 0, 0, 0); means the color red and transparent.

CSS border

There are 4 properties that can help in setting up the border:

  • Width
    • sets the width of the border
  • Style
    • sets the style of border; e.g. solid, dashed, etc.
  • Color
    • sets the color of the border
  • Radius
    • determines the roundness of the border

    Borders can be set specifically for top, right, bottom, and left. The top and bottom can be clubbed together as well as the left and right. E.g.: border-width: 2px 5px; (sets top and bottom to 2px, with left and right set to 5px). Borders can also be set in a single line. E.g.: border: 2px solid black;

CSS box model

Every element is CSS can be represented using the BOX model, which allows a border and defined space between the content.


Helping the developer to develop and manipulate the elements, it consists of 4 edges:

  • Content edge
    • comprising of the actual content
  • Padding edge
    • padding lies in between content and border edge
  • Border edge
    • the border follows padding
  • Margin edge
    • this is an outside border and controls the margin of the element

    Tip: In the Google Chrome browser, right-click and click "Inspect". This will display all the elements used in the HTML file. On the right-hand side, there will be a box model displayed, with the CSS parameters indicated. The most inner rectangle represents the content, the padding is the space between the content and the border, with the margin area outside the border.

CSS comments

CSS comments are ignored by browsers and are therefore not displayed on the web page. They are added to help understand the code and make it readable, as well as helping to debug. It is also helpful when editing the source code at a later date.


A CSS comment is placed inside the <style> element, and starts with /* and ends with */:


Example


/* This is a single-line comment */
p { color: red; }

You can add comments whenever you want in the code:


Example


p { color: red; /* Set text color to red */ }

Comments can also span multiple lines:


Example


/* This is a multi-line
comment */
p { color: red; }
Summary

CSS (Cascading Style Sheets) is a design language that makes a website look more appealing than just plain text. Whereas HTML largely determines textual content, CSS determines visual structure, layout, and aesthetics. HTML is a markup language, and CSS is a style sheet language. Think "look and feel" when you think CSS.


If HTML were the engine components of a car, CSS would be the body style and the paint job. A website can run without CSS, but it certainly isn’t pretty.


CSS makes the front-end of a website shine and it creates a great user experience. Without CSS, websites would be less pleasing to the eye and likely much harder to navigate. In addition to layout and format, CSS is responsible for font color and more.


The benefits of CSS include:

  1. Faster Page Speed – More code means slower page speed. CSS enables you to use less code. CSS allows you to use one CSS rule and apply it to all occurrences of a certain tag within an HTML document.
  2. Better User Experience - CSS not only makes web pages easy on the eye, it also allows for user-friendly formatting. When buttons and text are in logical places and well organized, user experience improves.
  3. Quicker Development Time - With CSS, you can apply specific formatting rules and styles to multiple pages with one string of code. One cascading style sheet can be replicated across several website pages. If, for instance, you have product pages that should all have the same formatting, look, and feel, writing CSS rules for one page will suffice for all pages of that same type.
  4. Easy Formatting Changes - If you need to change the format of a specific set of pages, it’s easy to do so with CSS. There’s no need to fix every individual page. Just edit the corresponding CSS stylesheet and you’ll see changes applied to all the pages that are using that style sheet.
  5. Compatibility Across Devices - Responsive web design matters. In today’s day and age, web pages must be fully visible and easily navigable on all devices. Whether mobile or tablet, desktop or even smart TV, CSS combines with HTML to make responsive design possible.

    6. CSS is critical when it comes to the overall presentation of a web page. It makes web developers’ lives much easier when it comes to formatting. Mastering CSS is a vital part of becoming a valuable asset and a solid programmer.
